# סוגרים חשבון

אפליקציית Web מקומית לניהול הוצאות של חברים באירוע.

## איך מפעילים

פתח את הקובץ:

`start-app.cmd`

הוא יפתח את האפליקציה בכתובת:

`http://127.0.0.1:4180`

## מה אפשר לעשות עכשיו

- לפתוח אירוע חדש.
- לחפש אירועים לפי שם במסך הבית.
- לבחור “מי אני עכשיו” לצורך בדיקה כמישהו אחר.
- להשתמש בקבוצה קיימת או באירוע חד-פעמי.
- ליצור קבוצות קבועות.
- לערוך שם, חברים ומנהלים בקבוצה קיימת.
- להוסיף חברים ואורחים.
- להגביל עריכת אירוע למנהלים בלבד.
- להוסיף הוצאה עם כמה משלמים.
- לבחור מי שותף לכל הוצאה.
- לערוך הוצאה קיימת.
- למחוק הוצאה אם טעו.
- ליצור קישור הזמנה לאירוע.
- לפתוח אירוע ישירות מקישור הזמנה.
- להצטרף לאירוע כאורח מתוך מסך האירוע.
- לראות מי צריך להעביר למי.
- להעתיק סיכום התחשבנות קצר לשליחה בוואטסאפ.
- לסמן העברה כשולמה.
- לבטל סימון שולם אם הייתה טעות.
- ליצור אירוע דומה מאירוע קיים בלי לגרור הוצאות קודמות.
- להעתיק דוח אירוע מלא עם הוצאות והתחשבנות פתוחה.
- לנעול אירוע לעריכה.
- לשמור את המידע דרך שרת מקומי לקובץ `data/app-state.json`.
- לייצא גיבוי JSON ולשחזר ממנו את כל המידע.
- לראות כתובת לפתיחה בטלפון שנמצא על אותו Wi-Fi.
- לראות במסך הבית האם האפליקציה מוכנה לשליחה לחברים כבטא ציבורית.
- להתחבר לשמירה משותפת ב-Supabase כשמגדירים משתני סביבה מתאימים.

## מה עדיין לא מחובר

- התחברות עם Google.
- שיתוף חי בענן בין כמה טלפונים מחוץ למחשב המקומי.
- שמירה בענן.
- פרסום לחנויות אפל/אנדרואיד.

הדברים האלה דורשים backend חיצוני וחשבונות/מפתחות מתאימים.
הכנה לבטא ציבורית נמצאת ב-`docs/beta-launch.md`.
אחרי הגדרת `.env` או משתני סביבה, אפשר לבדוק את מצב הפריסה דרך `/api/health`.
הפרויקט כולל גם `vercel.json`, כך שאפשר לפרוס אותו ל-Vercel ולקבל כתובת ציבורית.

## הערה על שמירה

אם מפעילים דרך `start-app.cmd`, האפליקציה עובדת מול השרת המקומי ושומרת את המידע בקובץ מקומי. אם פותחים את `index.html` ישירות, האפליקציה עדיין תעבוד, אבל תשתמש בשמירה של הדפדפן בלבד.

## פתיחה בטלפון באותו Wi-Fi

פתח את `start-app.cmd`, ואז במסך הבית תופיע כתובת רשת מקומית אם קיימת. פתח את הכתובת הזאת בטלפון שנמצא על אותו Wi-Fi.

אם Windows מבקש אישור Firewall, צריך לאשר גישה לרשת פרטית כדי שטלפון יוכל להתחבר.

## התקנה למסך הבית

האפליקציה כוללת עכשיו בסיס PWA: קובץ התקנה לדפדפן, אייקון, צבע נושא ושמירה בסיסית של קבצי האפליקציה.

באנדרואיד אפשר לפתוח את הכתובת בדפדפן ולבחור "Add to Home screen" אם הדפדפן מציע. באייפון אפשר להשתמש ב-"Share" ואז "Add to Home Screen".

חשוב: התקנה אמינה בטלפונים, במיוחד מחוץ למחשב המקומי, דורשת כתובת HTTPS ציבורית. זה יהיה השלב שבו נחבר פריסה אמיתית לענן.
